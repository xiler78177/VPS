# CDN 优选 IP 运维手册

> 一句话：国内机定时跑 CloudflareSpeedTest，按节点地区码优选 IP，直接生成本地节点文件；需要的话再同步入口域名 DNS。

---

## 一、角色

| 设备 | 作用 |
|---|---|
| 落地 VPS | 提供 Reality + CDN 回源链路 |
| 国内优选机 | 跑 `preferip-collect.sh` + `preferip-push.sh` |
| 客户端/分发程序 | 读取 `preferip.rendered.txt` 或入口域名 |

---

## 二、目录

| 文件 | 作用 |
|---|---|
| `nodes.txt` | 节点清单，推荐 `备注|CF地区码|vless链接` |
| `cdn-preferip.conf` | 优选参数、输出文件、DNS 配置 |
| `preferip-collect.sh` | 跑 CloudflareSpeedTest，生成结果文件 |
| `preferip-push.sh` | 生成本地节点文件，可选同步 DNS |
| `preferip-cron.sh` | 串联 B + C，给 cron 用 |
| `preferip.result` | 优选结果 |
| `preferip.rendered.txt` | 最终输出节点文件 |

---

## 三、最常用流程

1. 在落地机生成 CDN 节点链接。
2. 把链接写进国内机 `nodes.txt`。
3. 配好 `cdn-preferip.conf`。
4. 手动跑一次：

```bash
cd /root/cdn-preferip
bash preferip-cron.sh
```

5. 客户端或分发程序读取 `preferip.rendered.txt`。

---

## 四、推荐配置

```ini
PREFERIP_OUTPUT_FILE="/root/cdn-preferip/preferip.rendered.txt"
CFST_TOP_N="3"
CFST_ROUNDS="3"
CFST_PICK_MODE="speed"
CFST_EXTRA_ARGS="-n 100 -dn 5 -dt 10 -tl 180 -tlr 0 -sl 1"
PREFERIP_ASSIGN_MODE="round_robin"
PREFERIP_STICKY="true"
MISSING_COLO_POLICY="keep"
KEEP_ON_EMPTY="true"
```

`round_robin + CFST_TOP_N>1` 时会优先把同地区节点分摊到多个候选 IP；sticky 只在单候选或非轮询模式下阻止无收益切换。

如果只想让客户端不依赖文件刷新，可以给节点加入口域名：

```ini
PREFERIP_SERVER_MODE="dns"
PREFERIP_CF_API_TOKEN="..."
PREFERIP_DNS_PROXIED="false"
```

节点示例：

```text
香港-1|HKG|entry=prefer-hkg.example.com|vless://...
香港-ipv6|HKG|ipv6|entry=prefer6-hkg.example.com|vless://...
```

注意：

- `entry=` 必须是独立入口域名，不能等于原节点里的 `server` / `host` / `sni` CDN 回源域名。
- 原 CDN 域名继续保持橙云并指向真实源站；入口域名保持 DNS only，脚本只改入口域名到优选 IP。
- `PREFERIP_SERVER_MODE=dns` 下节点没写 `entry=` 会直接中止，避免误改原橙云记录。
- IPv4 与 IPv6 节点可以共用同一个 `entry=`；脚本会同时维护 A + AAAA，不会互删。

## 五、排查

| 现象 | 看什么 |
|---|---|
| 没生成输出文件 | `preferip.result` 是否为空，`PREFERIP_OUTPUT_FILE` 是否可写 |
| 某地区没结果 | 先放宽 `CFST_EXTRA_ARGS`，或把地区码改成实际可用的 colo |
| IPv6 没生效 | 检查 `CFST_IPV6_FILE` 和节点里的 `ipv6` 标记 |
| DNS 没更新 | 检查 `PREFERIP_SERVER_MODE=dns`、Token、zone id |
