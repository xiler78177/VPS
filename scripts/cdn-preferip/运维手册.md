# CDN 优选 IP — 运维手册

> 一句话原理:每台 VPS 建好 CDN 节点后,把节点的 vless 链接填进**优选机**的 `nodes.txt`;
> 优选机每天自动选出"你→CF边缘"质量最好的 CF IP,替换节点链接里的 server,推到 sub-store;客户端刷订阅即用。
> 默认可按 `nodes.txt` 里的 CF 地区码分组优选；如果实测亚洲方向只有 HKG 稳定，也可以把亚洲节点统一写成 `HKG`。

---

## 一、系统全貌(谁在哪、干什么)

| 设备 | 地址 / 私钥 | 角色 |
|---|---|---|
| **落地 VPS**(多台) | 各自 | 用 Reality 菜单 **10** 建 CDN 节点;负责回源出网 |
| **优选机 Ubuntu** | `国内机IP:SSH端口` / `运维用户` | 每天优选 CF IP + 刷 sub-store。脚本目录 `/root/cdn-preferip/` |
| **sub-store** | `sub.example.com` | 存专用订阅 `cdn-preferip` |
| **客户端**(软路由 OpenClash) | `客户端网关IP` | 经 proxy-providers 拉 collection(含 cdn-preferip) |

**数据流**:
```
落地VPS建节点 → 复制vless链接 → 填进优选机 nodes.txt
   → cron每天按地区/策略优选IP、替换每条链接的server → PATCH sub-store专用订阅 → 客户端刷订阅
```

**关键文件(都在优选机 `/root/cdn-preferip/`)**:
| 文件 | 作用 | 要不要手动改 |
|---|---|---|
| `nodes.txt` | **节点清单**,推荐每行 `区域备注\|CF地区码\|vless链接` | ★ 加/删机器就改这里 |
| `cdn-preferip.conf` | sub-store 地址、CFST 路径等 | 一般不动 |
| `preferip-cron.sh` | 串联:优选 + 刷订阅(cron 调它) | 不动 |
| `preferip.result` | 当天选出的优选 IP | 自动生成,不用管 |
| `preferip.state.tsv` | 每个节点上次采用的 IP/指标,用于稳态切换 | 自动生成,不用管 |
| `preferip.history.csv` | 候选/应用/失败历史记录 | 排查时查看 |
| `bad-ip.txt` | 探活失败等临时黑名单 | 排查时查看 |
| `backup/` | 每次 PATCH 前的 sub-store 内容备份 | 需要回滚时用 |
| `/opt/cfst/cfst` | CloudflareSpeedTest 二进制 | 不动 |

## 二、★ 加一台 VPS(最常用操作)

**第 1 步 — 在新 VPS 上建 CDN 节点**
进 Reality 菜单 → `10. 加挂 CDN 链路` → 按提示填 CF Token / CDN 子域名 → 完成后**复制屏幕打印的那条 `vless://...` 链接**。

**第 2 步 — 把链接填进优选机**(从你电脑)
```bash
ssh -i ~/.ssh/你的密钥 -p SSH端口 root@国内优选机IP
nano /root/cdn-preferip/nodes.txt
```
在文件末尾**加一行**(推荐格式 `区域备注|CF地区码|链接`,备注就是客户端里显示的名字；IPv6 优选池用 `区域备注|CF地区码|ipv6|链接`):
```
香港-xxx-1-CDN|HKG|vless://粘贴你刚复制的整条链接
韩国-xxx-1-CDN|ICN|vless://粘贴你刚复制的整条链接
日本-xxx-1-CDN|NRT,KIX|vless://粘贴你刚复制的整条链接
香港-ipv6-xxx-CDN|HKG|ipv6|vless://粘贴你刚复制的整条链接
```
> 备注随你写,建议 `地区-机器名-序号-CDN`;中间的 CF 地区码用于按地区优选,链接里 server 填什么都行,会被优选 IP 覆盖。
> 加第二台就再加一行,删机器就删那行,行首 `#` 是注释。
> 如果韩国/日本等地区测不到可用 CF edge，而 HKG 实测稳定，可直接写成 `韩国-xxx-1-CDN|HKG|vless://...`，让亚洲节点统一走 HKG 优选。

**可选：提高筛选标准**
在 `/root/cdn-preferip/cdn-preferip.conf` 里可设置：
```ini
CFST_EXTRA_ARGS="-n 100 -dn 5 -dt 10 -tl 180 -tlr 0 -sl 1"
CFST_TOP_N="3"
CFST_ROUNDS="3"
CFST_PICK_MODE="speed"
MISSING_COLO_POLICY="keep"
CFST_IPV6_FILE="/opt/cfst/ipv6.txt"
PREFERIP_ASSIGN_MODE="round_robin"
PREFERIP_STICKY="true"
```
含义：降低并发、启用下载测速、平均延迟低于 180ms、零丢包、下载速度至少 1MB/s；每个地区保留 Top3，多跑 3 轮后优先按速度挑选，并把同地区多个节点轮询分配到不同候选 IP。某个地区没结果时，该节点保留原 server，其它节点照常更新。

**可选：二阶段复测 / 真实链路探活**
```ini
CFST_STAGE2_ENABLE="true"
CFST_STAGE2_TOP_N="20"
CFST_STAGE2_ROUNDS="2"
PREFERIP_PROBE_ENABLE="true"
```
二阶段会先粗筛候选 IP，再只对候选做复测；探活会在回写前用 `curl --resolve` 验证“优选 IP + SNI/Host + path”是否能走通，失败 IP 会临时写入 `bad-ip.txt`。

**第 3 步 — 立即生效并验证**
```bash
cd /root/cdn-preferip && bash preferip-cron.sh      # 优选+刷,约 3-4 分钟
# 看订阅里现在有哪些节点(备注+server):
source cdn-preferip.conf
curl -s "$SUBSTORE_BASE/api/sub/cdn-preferip" | jq -r '.data.content'
```
看到新节点(备注对、server 是个 IP)就成了。**客户端刷新订阅**即可用。不手动跑也行,次日 05:17 cron 会自动带上。

## 三、删 / 改节点
```bash
ssh -i ~/.ssh/你的密钥 -p SSH端口 root@国内优选机IP
nano /root/cdn-preferip/nodes.txt        # 删行=删机器;改 | 左边=改备注名
cd /root/cdn-preferip && bash preferip-cron.sh   # 立即生效(或等次日自动)
```

## 四、手动立即刷一次优选(不等 cron)
```bash
ssh -i ~/.ssh/你的密钥 -p SSH端口 root@国内优选机IP
cd /root/cdn-preferip && bash preferip-cron.sh
```

## 五、看日志 / 排查
```bash
ssh -i ~/.ssh/你的密钥 -p SSH端口 root@国内优选机IP
tail -30 /var/log/cdn-preferip.log       # cron 历史(优选IP、PATCH结果)
crontab -l | grep preferip               # 确认定时任务还在(应是 17 5 * * *)
cat /root/cdn-preferip/preferip.result   # 当前优选IP
tail -20 /root/cdn-preferip/preferip.history.csv  # 最近候选/应用历史
cat /root/cdn-preferip/preferip.state.tsv          # 每个节点当前采用的 IP
cat /root/cdn-preferip/bad-ip.txt                  # 临时跳过的坏 IP
```
常见问题:
- **某节点变慢/绕远** → 先检查 `nodes.txt` 中间的 CF 地区码是否符合节点落地位置,例如香港用 `HKG`,韩国用 `ICN`,日本可用 `NRT,KIX`。
- **某地区长期测不到 IP** → `MISSING_COLO_POLICY=keep` 会保留该节点原 server、不影响其它节点更新；也可以临时把该节点地区码改成实测稳定的 `HKG`。
- **多个节点同时被一个优选 IP 拖慢** → 把 `CFST_TOP_N` 调到 `3` 或更高，并保持 `PREFERIP_ASSIGN_MODE=round_robin`，同地区节点会轮询分配不同候选 IP。
- **优选 IP 波动很大** → 保持 `PREFERIP_STICKY=true`；只有新 IP 在速度或延迟上明显更好时才切换，减少“频繁换但体验变差”。
- **CFST 结果快但实际连不上** → 开启 `PREFERIP_PROBE_ENABLE=true` 做真实链路探活；失败 IP 会临时写进 `bad-ip.txt`。
- **某台节点要用 IPv6 优选池** → XIU2/CloudflareSpeedTest 自带 `ipv6.txt`;在 `cdn-preferip.conf` 设置 `CFST_IPV6_FILE="/opt/cfst/ipv6.txt"`，并把该节点写成 `备注|HKG|ipv6|vless://...`。
- **某节点连不上** → 多半是那台落地机本身/证书问题,不是优选;先在客户端单测该节点延迟。
- **优选IP换了客户端还旧** → 客户端要**刷新订阅**(OpenClash 更新那条订阅)。
- **优选机关机了** → cron 不跑、IP 会过期。优选机(Ubuntu)需长期开机。

## 六、关键约束(别踩)
1. **优选必须在国内直连网络的机器上跑** —— 如果优选机自身走代理，测出的 CF IP 对客户端无意义，甚至可能连不上。
2. **优选机要保持开机**,否则每天的自动优选不执行。
3. 日常只动 `nodes.txt`;`SUBSTORE_BASE` 等在 `cdn-preferip.conf`,改了 sub-store 地址才需要动。
